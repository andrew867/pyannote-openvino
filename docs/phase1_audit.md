## ONNX export attempts
The audit script also tries to export both models to ONNX (saved under `artifacts/`). Both exports currently fail in `torch.export.export` because `dynamic_shapes` must only reference argument names and cannot include intermediate outputs such as `scores` or `embeddings`. The captured exceptions (see the `onnx` section of the summary file) read:

- Segmentation: `torch._dynamo.exc.UserError` complaining that `dynamic_shapes` keys must be argument names (`waveforms` only, not `scores`).
- Embedding: same type of error with `['waveforms', 'embeddings']`.

The full logs are in `artifacts/phase1_summary.json`; the script logs also show these problems in the console (with the same instructions from PyTorch). The exports therefore require revisiting in Phase 2 (for example: supply `dynamic_shapes` as a tuple/list or wrap the models so the exporter only sees the waveform input and one output).

## Notes
- Importing `pyannote.audio` emits a warning that `torchcodec` cannot load prebuilt FFmpeg-based DLLs (torchcodec bundles `libtorchcodec_core*.dll`). This can be ignored for the audit because we provide waveform tensors directly, but it explains why the pipeline warns about audio decoding.
